wget 类似于迅雷，是一种下载工具，通过HTTP、HTTPS、FTP三个最常见的TCP/IP协议 下载，并可以使用HTTP代理名字是World Wide Web”与“get”的结
合。
yum: 是redhat, centos 系统下的软件安装方式，基于Linux，全称为 Yellow dog Updater, Modified，是一个在Fedora和RedHat以及CentOS中的Shell前端软件包管理器基于RPM包管理，能够从指定的服务器自动下载RPM包并且安装，可以自动处理依 赖性关系，并且一次安装所有依赖的软件包。
rpm:软件管理;redhat的软件格式rpm r=redhat p=package m=management 用于安装 卸载 .rpm软件

0x02:2016.04.19 周二 晴
Find、grep、sed、awk使用方法
find 用于查找文件系统中的指定文件
find . -name 1.txt （.当前目录 ..上级目录）查找名字为1.txt（遍历查找）
-name “1.”使用通配符需要加上双引号或者单引号可以再任何位置
find /etc/ -name .conf print 查找的文件输出出来
find /tmp -name ‘.*’查找/tmp目录下的所有文件
find /bin/ -perm 4755 | xargs ls -al 把前一个命令的输出转化为后一条命的参数
find /bin/ -perm 4755 -exec ls -al {} \; ls -al命令放在find命令的-exec选项中

grep文本搜索工具，使用正则表达式搜索文本并打印出来
-c：只显示匹配行的行数计数
-i：匹配时不区分大小写，默认情况下匹配时区分大小写
-h：在多个文件中查找时，输出行前不显示文件名
-n：在出前加上匹配串所在行的行号（文件首行行号为1）
-v：只显示不包含匹配串的行。
-f filename：从指定文件中获取搜索模式，每行一个搜索模式项
-A匹配行以及下面的3行grep -A 3 “root” /etc/passwd –color
-B匹配行以及上面的3行grep -B 3 “root” /etc/passwd –color
-C匹配行以及上下的3行grep -C 3 “root” /etc/passwd –color
grep 文件名 目录 在目录下查找文件
grep -nr “^root” /etc/ –color 以root开头的
ps -ef | grep ora通过ps显示正在活动的进程重定向输出（查找ps -ef 中有没有ora串）
Grep -n root /etc/passwd –color
Grep -vn root /etc/passwd
grep -vn root /etc/passwd | grep -v nologin

sed 文本处理工具，本身是一个管道命令主要以行为单位进行处理，可以将数据进行增删改选：对文件的编辑！
-n:使用安静模式。
-e:直接在命令行模式运行sed的动作编辑（可以有多个动作）
-f:直接将sed的动作写在一个文件里边：写入一个新的文件（很少用）
-r:sed的动作支持的是延伸正规表示的语法
-i:覆盖（直接诶修改读取的文件类型）
动作说明：
a:增加
c:取代
d:删除
i插入
p:打印
s:取代
nl /etc/passwd | sed ‘2,$d’ nl输出到管道里边 操作的是缓存
nl /etc/passwd | sed ‘2a jiadong’ 新增向上
nl /etc/passwd | sed ‘2i aaaaaa’插入是向下插入
nl /etc/passwd | sed ‘2,5c aaaaa’将2-5行替换为aaaaa
nl /etc/passwd | sed -n ‘2,5p’安静输出2-5行
nl /etc/passwd | sed -n ‘/root/p’根据正则表达式列出输出的东西
nl /etc/passwd | sed ‘/root/d’匹配到root的那一行删除掉
nl /etc/passwd | sed -n ‘/root/p’
nl /etc/passwd | sed -n ‘/root/{s/root/abcd/g;p}’只在有/root的行里边替换root输出abcd /g全局替换
nl /etc/passwd | sed -n ‘1,7{s/root/abcd/;p}’1-7行里边替换开头/root

awk是一个强大的文本分析工具，相对于grep的查找，sed的编辑，awk在其对数据分析并生成报告时，显得尤为强大。简单来说awk就是把文件逐行的读入，以空格为默认分隔符将每行切片，切开的部分再进行各种分析处理。

将每行切片：对文件的抽取！
每读文件一行就执行命令
last -n 5 | awk ‘{print $1}’输出前五行的第一列
last -n 5 | awk ‘{print”name:”$2}’
last -n 5 | awk ‘{print”name:”$2”\tip:”$3}’
cat /etc/passwd | awk -F ‘:’ ‘{print “usernaem: “$1}’
print “%-20s”美化格式

服务与计划任务：

服务脚本里边的一些配置文件运行真正程序是加载配置文件

service httpd start
/etc/init.d/httpd start 连接到/etc/rc.d/init.d调用服务！属于软连接
进入 显示各种服务：
显示使用方法：
添加服务 放脚本到：
/etc/init.d/目录下
Chkconfig 查看服务
chkconfig –list 列出服务列表

计划任务（crond 来实现定时任务的完成）
cron把命令行保存在/etc/crontab文件里，每个系统用户如果设置了自己的cron，那都会在/var/spool/cron下面有对应用户名的crontab。无论编写/var/spool/cron目录内的文件还是/etc/crontab文件，都能让cron准确无误地执行安排的任务，区别是/var/spool/cron下各系统用户的crontab文件是对应用户级别的的任务配置，而/var/crontab文件则是对应系统级别的任务配置。cron服务器每分钟读取一次/var/crontab/cron目录内的所有文件和/etc/crontab文件。
cron也是一个服务
每分钟醒来一次查看当前是否需要运行命令
运行周期一分钟
添加计划任务先去/var/spool/cron
两种添加计划任务
1、crontab -e
/1**/root//
2、编辑/etc/crontab(必须要有root用户权限)
/1**/root//

用户计划任务的格式
添加计划任务
crontab -e编辑/1 run-parts /etc/cron.hourly

PATH=/sbin:/bin:/usr/sbin:/usr/bin 环境变量
