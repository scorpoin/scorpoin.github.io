How to use the grep command in Linux

0x00介绍
Grep文本搜索工具，使用正则表达式搜索文本并把匹配的行打印出来

0x01主要参数：
-a: 将二进制（binary）文件以text文件的方式搜索
-c：只显示匹配行的行数计数
-i：匹配时不区分大小写，默认情况下匹配时区分大小写
-h：在多个文件中查找时，输出行前不显示文件名
-n：在输出前加上匹配串所在行的行号（文件首行行号为1）
-v：只显示不包含匹配串的行。
-r递归目录下所有目录
-f filename：从指定文件中获取搜索模式，每行一个搜索模式项
-A匹配行以及下面的3行grep -A 3 “root” /etc/passwd –color
-B匹配行以及上面的3行grep -B 3 “root” /etc/passwd –color
-C匹配行以及上下的3行grep -C 3 “root” /etc/passwd –color

0x02正则表达式中的主要参数：
\:转意符
^：匹配正则表达式的开始行
$：匹配正则表达式的结束行
\<:从匹配正则表达式的行开始
>:到匹配正则表达式的行结束

。：所有字符串
*：所有字符
Grep 文件名 目录 在目录下查找文件

0x03实例：
grep -nr “^root” /etc/ –color 以root开头的

grep root /etc/passwd –color将有root的行打印出来

gerp -v root /etc/passwd 输出不包含root的行

ps -ef | grep ora通过ps显示进程重定向输出包含查找字符的文件

grep -v root /etc/passwd | grep -v nologin将不包含root和nologin行输出出来

dmesg | grep -n ‘JD’ /etc/passwd输出重要的信息‘信息’

0x04grep中使用正则表达式
grep -n ‘xiao*’ jia 搜索字符注意：单引号双引号均可；

grep -nr ‘^x’ jia在文件中搜索以字符x开头的

不解释（反向搜索，不包含a-j的）

0X05拓展：
为了使每次执行grep的时候添加颜色而不去输入–color 可以修改配置文件
vim ~/.abshrc在末尾添加alias grep=’grep –color=auto’需要重启一下就可以了
