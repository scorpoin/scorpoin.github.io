VPS搭建Metasploit详细教程

一篇详细的vps安装metasploit实例

Metasploit是一款强大的开源安全漏洞检测工具，攻击渗透功能十分强大，具有丰富的exp以及payload,在kali以及更早版本的bt系列的系统上已经集成了metasploit，工作人员常用 Metasploit 工具来检测系统的安全性。Metasploit Framework (MSF) 在 2003 年以开放源码方式发布，是可以自由获取的开发框架。它是一个强大的开源平台，供开发，测试和使用恶意代码，这个环境为渗透测试、shellcode 编写和漏洞研究提供了一个可靠平台。今天在一台外网vps上边安装一下，以便日后方便使用，安装过程会详细记录，按照下述步骤保证你成功安装，安装步骤已经过多次测试。
安装环境：
外网的一台vps，系统版本信息如下：
[root@VM_9_220_centos ~]#uname -a
Linux VM_9_220_centos 2.6.32-504.30.3.el6.x86_64 #1 SMP Wed Jul 15 10:13:09 UTC 2015 x86_64 x86_64 x86_64 GNU/Linux

###【详细安装流程】
一、必要开发环境
1、更新yum源，安装metasploit需要安装一系列的依赖包，更新yum源是很有必要的！
[root@VM_9_220_centos ~]#yum update

2、安装开发环境依赖包，理由同上：
[root@VM_9_220_centos ~]#yum groupinstall 'development tools'

3、安装msf所需要的开发工具依赖包：
[root@VM_9_220_centos ~]#yum install sqlite-devel libxslt-devel libxml2-devel java-1.7.0-openjdk libpcap-devel nano openssl-devel zlib-devel libffi-devel gdbm-devel readline-devel nano wget

上述依赖包以及开发工具安装完成之后接下来就需要安装ruby环境，因为新版本的metasploit开发人员是用ruby写的，最开始的时候metasploit是用perl脚本语言写的，一直到2006年的2.7版本一直都还是使用perl语言编写的，后来开发人员发现perl存在一些缺陷，所以在2007年底使用ruby语言重写了metasploit的框架。

二、相关组件安装

1、RVM安装

5、安装ruby之前先安装一下ruby的管理工具RVM，个人推荐使用rvm安装ruby使用方便，使用curl下载rvm：
[root@VM_9_220_centos ~]#curl -L get.rvm.io | bash -s stable

安装过程中会出现失败，不要慌，按照提示进行一步一步安装：

需要下载一个验证码之类的东西，下载完成之后在执行安装rvm
[root@VM_9_220_centos ~]#curl -sSL https://rvm.io/mpapis.asc | gpg2 --import -
[root@VM_9_220_centos ~]#curl -L get.rvm.io | bash -s stable

当出现Thank you for using RVM!证明安装完成了：

找到rvm的配置文件，然使其生效：
[root@VM_9_220_centos ~]# find / -name rvm.sh
/etc/profile.d/rvm.sh
[root@VM_9_220_centos ~]# source /etc/profile.d/rvm.sh

其实到这里已经可以使用rvm进行ruby的安装及使用了，为了后续的完美使用，我们现在装一下rvm所需要的一些依赖：
[root@VM_9_220_centos ~]# rvm requirements
Checking requirements for centos.
Installing requirements for centos.
Installing required packages: libyaml-devel……
Requirements installation successful.
下边检查一下看看少装依赖了没有：
[root@VM_9_220_centos ~]# rvmsudo yum install -y gcc-c++ patch readline readline-devel zlib zlib-devel libyaml-devel libffi-devel openssl-devel make bzip2 autoconf automake libtool bison iconv-devel

上述rvm已经安装完成。

2、yaml安装

6、后续的一些组件在安装的时候需要源码安装，所以源码安装之前需要先安装yaml，yaml是 表达资料序列的编程语言
[root@VM_9_220_centos ~]#cd /usr/src //切换到src目录下
[root@VM_9_220_centos src]#wget http://pyyaml.org/download/libyaml/yaml-0.1.6.tar.gz //wget下载压缩包
[root@VM_9_220_centos src]#tar -zxvfyaml-0.1.6.tar.gz //解压压缩包
[root@VM_9_220_centos src]# rm -rf yaml-0.1.6.tar.gz //删除压缩包
[root@VM_9_220_centos src]#cd yaml-0.1.6 //切换到解压包中
[root@VM_9_220_centosyaml-0.1.6]#./configure--prefix=/usr/local //将可执行脚本执行到指定目录中–prefix指定安装路径
[root@VM_9_220_centosyaml-0.1.6]#make && make install //编译、安装
[root@VM_9_220_centos yaml-0.1.6]# make clean //清除缓存文件
安装完成！

三、安装nmap

7、据说metasploit中没有nmap那就安装一下吧，namp是一款强大的端口扫描工具，在渗透测试过程中，情报收集阶段利用非常方便：
会在当前目录下， 增加一个 nmap文件夹； svn的namp文件的内容， 在nmap文件夹中
[root@VM_9_220_centos src]# svn co https://svn.nmap.org/nmap
[root@VM_9_220_centos src]# cd nmap
[root@VM_9_220_centos nmap]# ./configure
[root@VM_9_220_centos nmap]# make
[root@VM_9_220_centos nmap]# make install
[root@VM_9_220_centos nmap]# make clean
安装完成：

四、安装ruby

8、Ruby 是一种面向对象、命令式、函数式、动态的通用编程语言
先查看一下安装的有没有ruby版本：
[root@VM_9_220_centos yaml-0.1.6]# rvm list

安装最新版本的ruby,metasploit貌似对ruby版本还有要求，网上说的是不能低于1.9.2，不管这些，安装爱装最新版本的就对了：
[root@VM_9_220_centos src]# rvm install ruby-2.3.1
[root@VM_9_220_centos src]# rvm list

五、安装postgresql

9、msf使用的是postgrepsql server的数据管理库， PostgreSQL 是一个自由的对象-关系数据库服务器(数据库管理系统)
首先修改一下配置文件：
[root@VM_9_220_centos src]# vim /etc/yum.repos.d/CentOS-Base.repo
增加的内容：exclude=postgresql*

接下来下载安装：
[root@VM_9_220_centos src]# wget http://yum.postgresql.org/9.2/redhat/rhel-6-x86_64/pgdg-centos92-9.2-6.noarch.rpm
[root@VM_9_220_centos src]# rpm -ivh pgdg-centos92-9.2-6.noarch.rpm

更新一下安装包文件：
[root@VM_9_220_centos src]# yum update
[root@VM_9_220_centos src]# yum install postgresql92-server postgresql92-devel postgresql92

将安装好的postgresql数据库进行配置：

[root@VM_9_220_centos src]#service postgresql-9.2 initdb
Initializing database: [ OK ]
[root@VM_9_220_centos src]# service postgresql-9.2 start
Starting postgresql-9.2 service: [ OK ]
[root@VM_9_220_centos src]# chkconfig postgresql-9.2 on

添加环境变量，使其生效：
[root@VM_9_220_centos src]# echo export PATH=/usr/pgsql-9.2/bin:\$PATH >> /etc/bashrc
[root@VM_9_220_centos src]# source ~/.bashrc

给postgresql数据库创建用户，设置密码：
[root@VM_9_220_centos src]# su - postgres
-bash-4.1$ createuser msf -P -S -R -D
Enter password for new role:
Enter it again:
-bash-4.1$ createdb -O msf msf
-bash-4.1$ exit

管理数据库的配置文件：
[root@VM_9_220_centos src]# vim /var/lib/pgsql/9.2/data/pg_hba.conf
local msf msf md5
host msf msf 127.0.0.1/8 md5
host msf msf ::1/128 md5

修改完配置文件，重启一下服务：
[root@VM_9_220_centos src]# service postgresql-9.2 start
Starting postgresql-9.2 service: [ OK ]

六、安装metasploit

首先安装msf所需要的ruby库
[root@VM_9_220_centos src]#gem install wirble pg sqlite3 msgpack activerecord redcarpet rspec simplecov yard bundler
安装完成：

下载metasploit框架，将其放到opt目录下：
[root@VM_9_220_centos opt]# git clone https://github.com/rapid7/metasploit-framework.git
已经OK了：

切换到metasploit-framework目录中安装gem,经过漫长的等待终于安装完成：
[root@VM_9_220_centos metasploit-framework]# bundle install

编辑msf的数据库配置文件，使msf启动的时候能够加载数据库
[root@VM_9_220_centos metasploit-framework]# nano /opt/metasploit-framework/database.yml
写入的内容：
production:
adapter: postgresql
database: msf
username: msf
password: 123456
host: 127.0.0.1
port: 5432
pool: 75
timeout: 5

添加环境变量使其生效：
[root@VM_9_220_centos metasploit-framework]# echo export MSF_DATABASE_CONFIG=/opt/metasploit-framework/database.yml >> /etc/bashrc
[root@VM_9_220_centos metasploit-framework]# source ~/.bashrc

接下来就是运行msfconle

成功安装，接下来连接数据库：
重启一下postgresql-9.2服务，成功连接数据库：

【总结】

前前后后加上在本地虚拟机上测试安装不下五次，安装过程好多细节问题需要注意，写一篇总结为的是记录一下，其中需要注意的是：
    1、ruby的版本问题，最好是安装最新的版本；
    2、数据库的配置；
    3、最后安装完成之后启动msf的时候直接输msfconle可能会提示找不到命令，那就试一下./msfconle，还不行的话就看一下文件的权限；
    4、运行msfconcle时提示连接数据库失败，那就退出来启动一下postgresql-x.x服务就行了
