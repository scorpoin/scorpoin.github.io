TCP/IP data transport

0x00背景

前一段时间学习了TCP/IP协议，今天把数据传输再理一下！

0x01协议​

首先要知道tcp/ip协议簇为五层协议

应用层：协议有SMTP FTP TFTP DNS SNMP HTTP

传输层：协议有TCP UDP SCTP

网络层：协议有IP ICMP IGMP ARP RARP

链路层：协议有SLIP PPP

物理层：

0x02传输过程

数据从发送到接收所经过的详细过程为：

发送方：

首先数据从应用层发出，在应用层寻找相应的协议，比如发送一封邮件，首先会在应用层找到对应的协议SMTP协议，在应用层会为邮件数据添加一个头部，头部内容为（收信、发信的邮箱地址等信息）

然后将数据传送到传输层，调用传输层的TCP协议，TCP在接收到数据之后会为数据增加一个头部，头部内容包括（源端口，目的端口）

数据传输至网络层，网络层IP协议又会为数据增加一个头部（源IP和目的IP）

数据传输至链路层，链路层为收到的数据增加一个源MAC和目的MAC，在数据的尾部增加一个尾部，保证数据完整性的数据校验和，

接下来传送至物理层，将接收到的二进制数据转化为模拟信号发送出去​​

接收方：

接收方在通过传输介质接收到模拟信号之后将模拟信号转化为数字信号（二进制数字），接下来就是进行层层的解封

首先数据到达物理层，紧接着是物理层会将数据提交到数据链路层

数据链路层收到数据之后会提取尾部校验和，对数据的完整性进行校验，完整的话就提取头部的MAC地址和自己的MAC地址表进行比对，比对成功之后去掉MAC头和校验尾向上提交，比对不成功的话就丢弃

数据到达网络层，网络层提取目的IP地址然后和自己的IP地址表进行比照，比照成功去掉IP头就进行向上提交，不成功就丢弃报文；

接着数据到达传输层，传输层提取端口，然后就将剩余的数据传到应用层，接收方这时就接收到了一封完整的邮件​

0x03总结

这只是简单的数据传输过程，数据的传输要经过层层的封装解封最终呈现出来​。
